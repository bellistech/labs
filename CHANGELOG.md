## How to Report Changes

When contributing updates, please categorize them:

### Added
New features, materials, or capabilities

### Changed
Modifications to existing materials for improvement

### Deprecated
Marking sections/approaches as outdated but still functional

### Removed
Removing outdated or incorrect materials

### Fixed
Corrections of bugs or inaccuracies

### Security
Important security-related updates

## Contributing to Changelog

When submitting changes:

1. Decide which section applies (Added, Changed, Fixed, etc.)
2. Write a clear, concise description
3. Include the course/section affected
4. Link to related issues if applicable

**Example:**
```markdown
### Added
- Kubernetes: New lesson on scheduling and affinity patterns
- Terraform: Working example for multi-environment deployments
- All courses: Setup validation scripts
```
